 # CineMax - Prueba de Maquetación Frontend

Este repositorio contiene una aplicación web de demostración (Single Page Application) construida desde cero. 

**Este proyecto ha sido desarrollado específicamente ** con el objetivo de demostrar mis habilidades técnicas, buenas prácticas y nivel de detalle como Desarrollador Frontend / Maquetador.

## Contexto del Proyecto

A lo largo de mi trayectoria profesional, he participado en proyectos de alto nivel para diversas empresas. Sin embargo, debido a estrictas políticas de confidencialidad interna y acuerdos de no divulgación (NDA) firmados con mis empleadores anteriores, me es imposible compartir enlaces directos, código fuente o repositorios de los productos en los que he trabajado.

Por esta razón, he decidido construir "CineMax", una maqueta funcional de una plataforma de streaming, para que puedan evaluar de primera mano la calidad de mi código, mi atención al detalle en UI/UX y mi dominio de las tecnologías base de la web.

## Tecnologías y Prácticas Utilizadas

Para esta demostración he optado por no utilizar ningún framework de terceros (como Bootstrap, Tailwind, React o Angular), demostrando así un dominio sólido y profundo de las bases:

- **HTML5 Semántico**: Estructura accesible y bien organizada.
- **CSS3 Vanilla (Diseño Responsivo)**:
  - Uso intensivo de **CSS Grid** y **Flexbox** para layouts complejos adaptables a cualquier dispositivo (Mobile First).
  - Implementación estricta de la metodología **BEM** (Block Element Modifier) para asegurar un código escalable y fácil de mantener.
  - Custom properties (Variables CSS) para un sistema de diseño (Design System) consistente.
- **JavaScript Moderno (ES6+)**:
  - Manipulación eficiente del DOM.
  - Consumo asíncrono de una API REST real (**The Movie Database - TMDB**) mediante `fetch`, `async/await` y `Promises`.
  - Arquitectura modular y limpia, separando la capa de datos (`api.js`) de la capa de presentación (`app.js`).
  - Implementación de estado de carga mediante *Skeleton Loaders*.

## Cómo ejecutar el proyecto

1. Clona este repositorio o descarga los archivos.
2. Abre el archivo `index.html` directamente en tu navegador web de preferencia. No se requiere instalación de dependencias ni levantar un servidor local para probar la interfaz principal.
